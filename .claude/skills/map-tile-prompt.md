# 元墟地图瓦片提示词技能

为小说《元墟》地图编辑页（MapEditor，L0 层 32×32 网格）的格子生成 AI 生图提示词。产出直接可复制去 Midjourney / Flux / 即梦等工具出图，图片回填 `public/maps/L0-{x}-{y}.jpeg`。

> 生成前先对齐世界观：`backgroundCards.js` 的 story-1（元墟星球）、story-4（元祇四族）；格子命名与已放图片见 `MapEditor/components/LeafletCanvas/data/demoWorld.js`。

## 核心铁律：高空俯瞰的宏观地貌口径

视野极高、覆盖数千平方公里，**一切地貌压缩成微小纹理**——只写色块、纹理、脉络、走向，不写任何近景单体（一棵树、一座峰、一栋房的细节都不许出现）。

| 地貌 | 固定比喻 |
|---|---|
| 山脉 | 像皱褶的布匹（脊线＋阴影褶皱） |
| 森林 | 像密布的苔藓（深绿绒面） |
| 河流 | 像蓝丝线（亮色脉络） |
| 城镇/聚落 | 像散落的沙粒 |
| 雾 | 像薄纱压成的细密褶皱 |
| 田野 | 几何拼色块、隐约格痕 |

微小元素一律用 **「针尖大 / 米粒大」** 量级词压尺寸（如"针尖大的晶柱立石"），防止 AI 画成近景大物。

## 一、固定模板

```
提示词 = 固定前缀 + 「地图内容是」 + 该格内容段
```

**固定前缀（逐字使用，不改动）：**

> 高空俯瞰的奇幻大陆地图，视野极高，覆盖数千平方公里，地貌被压缩成极小的纹理：山脉像皱褶的布匹，森林像密布的苔藓，河流像蓝丝线，城镇像散落的沙粒。整体风格为精细手绘淡彩，所有地理元素都极度密集且尺寸微小，复古制图美学，无空白区域，满幅无白边，边缘无缝衔接便于拼接，无边框，无文字，无网格，1:1正方形，超精细

**固定负面词：**

> 透视、地平线、天空、人物、建筑特写、单体树木特写、文字水印、边框留白、现代元素

## 二、内容段写法

1. **一段到底**：一个格子一段话，顿号/逗号连缀，不分行不列点。
2. **先主体后细节**：第一句定这格的主体地貌与占比（"深蓝海面为主体""连绵雪山占据整幅画面"），再叠 2~4 个宏观细节。
3. **方位衔接（拼接的关键）**：格子坐标 x 小=西、x 大=东、y 小=北、y 大=南。内容段里必须写清与邻格衔接的方位词——"北缘接山麓灰绿""东缘渐入苔原""数条河流自画面四缘向心汇入"，且须与邻格提示词的对应边一致。
4. **颜色成带分层**：海色"由深至浅分层"、湖色"由岸缘碧绿层层深至渊心墨黑"，用渐变词而非单色词。
5. **收尾点氛围**（可选）：一句气质词收束，如"肃穆神圣、亘古沉睡之感""朦胧有序、身处巨大意识之中的感觉"。

## 三、世界观锚点：中天温稳带 · 元祇大陆

大陆布局四祇分踞（新增格子先对号入座再写内容）：

| 方位 | 势力 | 地貌母题 |
|---|---|---|
| 北 | 玄珉（石之祇） | 雪山山系、高原台地、峡谷、风蚀垣、晶青地脉线 |
| 西 | 青荄（木之祇） | 无边林海、丘泽、青金色发光根脉 |
| 中/南 | 沄澜（水之祇） | 元渊圆湖、向心水系、河谷牛轭湖、三角洲、泽国 |
| 东 | 翳霭（气之祇） | 乳白雾原、螺旋雾漩、台地、盐泽 |
| 中偏东南 | 焚京劫墟 | 环状焦痕、放射裂纹、废城网格残迹、余烬暗橙 |
| 中部 | 故京平野 | 田垄格痕、河曲、湖沼碎镜 |
| 四缘 | 汐海 | 深蓝海、群岛环礁、浪沫环带 |

## 四、圣地规则（神圣・神秘）

四祇圣居与永恒之心是地标格，神迹必须转成**远观仍可读的大尺度几何纹理**：

| 圣地 | 远观几何 |
|---|---|
| 玄珉·岳主峰 | 放射状八脊如袈裟褶皱＋两重环状山岭成坛城，晶青光线四缘汇心 |
| 青荄·世树 | 巨型圆冠占画面近半、放射裂瓣如绿色漩涡，发光根脉伸向四缘，九处空地成年轮环 |
| 沄澜·元渊渊心 | 正圆深湖＋渊心闭眼、同心涟漪刻满湖面，水下螺旋沉阶暗纹，百川向心 |
| 翳霭·翳垆雾台 | 全幅雾海呈巨大缓慢螺旋，漩心圆台地刻三重同心石环，上空淡金光晕如巨瞳半睁 |
| 永恒之心（纯净核） | 正中一枚至亮白色小太阳＋同心光环，四方地貌（北雪脊/西根脉/南水环/东雾旋）向心汇聚咬合成四色坛城封环 |

要点：神圣感 = 同心圆、放射线、坛城、年轮这类**规整几何** 出现在天然地貌里；神秘感 = 发光脉络、反常向心（百川倒流、雾漩不散）、"巨物沉眠"暗示。

## 五、落地流程

1. 定格子坐标与归属分区 → 写内容段 → 拼前缀输出。
2. 图片存为 `public/maps/L0-{x}-{y}.jpeg`（cellId 命名）。
3. `demoWorld.js` 的 `IMAGES` 加一行 `'L0-{x}-{y}': '/maps/L0-{x}-{y}.jpeg'`，需要名字则在 `ZT_NAMES` / `Object.assign` 段补格子名（神话式命名，参考现有：珉西岭、荄北林、澜尾河谷、翳垆雾台）。
4. Vite HMR 自动刷新，无需重启。

## 使用方式

```
/map-tile-prompt [格子坐标或地貌/圣地描述]
```

示例：
- `/map-tile-prompt L0-16-16 焚京劫墟心`
- `/map-tile-prompt 大陆西南角的海湾森林格`

## 输出格式

每格输出一个完整可复制块（前缀＋内容段连成一段），块后附一行负面词；多格时按坐标逐格给出，并在末尾提示各格间的衔接边。

> 地图内容是深蓝海面为主体，东南角散布一串破碎群岛与环礁，岛缘白色浪沫环带，海色由深至浅分层

## 避坑

1. **禁近景**：出现"一棵/一座/一栋＋细节描写"就是写错了，改成纹理与量级词。
2. **前缀不许改写**：风格统一全靠前缀逐字一致，改一处则整批图风格漂移。
3. **衔接边先查邻格**：写某格前先看四邻已有的提示词/图片，边缘地貌必须对得上；邻格是海，本格该边就得是岸或海。
4. **圣地不堆玄虚词**：不写"神力涌动、仙气缭绕"这类空词，神圣感只靠几何纹理与反常地貌呈现（对齐能力设计的写实原则）。
5. **格子名与图片名分离**：提示词里绝不出现文字标注（无文字），格子名只写进 `demoWorld.js`。
